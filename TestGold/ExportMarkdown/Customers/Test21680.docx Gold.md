[^4]There[^5] are five different kinds of *elements*: [void elements](https://www.w3.org/TR/html5/syntax.html#void-elements), [raw text elements](https://www.w3.org/TR/html5/syntax.html#raw-text-elements), [escapable raw text elements](https://www.w3.org/TR/html5/syntax.html#escapable-raw-text-elements), [foreign elements](https://www.w3.org/TR/html5/syntax.html#foreign-elements), and [normal elements](https://www.w3.org/TR/html5/syntax.html#normal-elements).

*Normal elements*

All other allowed [HTML elements](https://www.w3.org/TR/html5/infrastructure.html#html-elements) are normal elements.

*Tags[^6]* are used to delimit the start and end of elements in the markup. [Raw text](https://www.w3.org/TR/html5/syntax.html#raw-text-elements), [escapable raw text](https://www.w3.org/TR/html5/syntax.html#escapable-raw-text-elements), and [normal](https://www.w3.org/TR/html5/syntax.html#normal-elements) elements have a [start tag](https://www.w3.org/TR/html5/syntax.html#syntax-start-tag) to indicate where they begin, and an [end tag](https://www.w3.org/TR/html5/syntax.html#syntax-end-tag) to indicate where they end. The start and end tags of certain [normal elements](https://www.w3.org/TR/html5/syntax.html#normal-elements) can be [omitted](https://www.w3.org/TR/html5/syntax.html#syntax-tag-omission), as described below in the section on [optional tags](https://www.w3.org/TR/html5/syntax.html#syntax-tag-omission). Those that cannot be omitted must not be omitted. [Void elements](https://www.w3.org/TR/html5/syntax.html#void-elements) only have a start tag; end tags must not be specified for [void elements](https://www.w3.org/TR/html5/syntax.html#void-elements). [Foreign elements](https://www.w3.org/TR/html5/syntax.html#foreign-elements) must either have a start tag and an end tag, or a start tag that is marked as self-closing, in which case they must not have an end tag.

The [contents](https://www.w3.org/TR/html5/dom.html#concept-html-contents) of the element must be placed between just after the start tag (which [might be implied, in certain cases](https://www.w3.org/TR/html5/syntax.html#syntax-tag-omission)) and just before the end tag (which again, [might be implied in certain cases](https://www.w3.org/TR/html5/syntax.html#syntax-tag-omission)). The exact allowed contents of each individual element depend on the [content model](https://www.w3.org/TR/html5/dom.html#content-models) of that element, as described earlier in this specification. Elements must not contain content that their content model disallows. In addition to the restrictions placed on the contents by those content models, however, the five types of elements have additional *syntactic* requirements.

[Void elements](https://www.w3.org/TR/html5/syntax.html#void-elements) can't have any contents (since there's no end tag, no content can be put between the start tag and the end tag).

[Raw text elements](https://www.w3.org/TR/html5/syntax.html#raw-text-elements) can have [text](https://www.w3.org/TR/html5/syntax.html#syntax-text), though[^7][^8] it has [restrictions](https://www.w3.org/TR/html5/syntax.html#cdata-rcdata-restrictions) described below.

[Escapable raw text elements](https://www.w3.org/TR/html5/syntax.html#escapable-raw-text-elements) can have [text](https://www.w3.org/TR/html5/syntax.html#syntax-text) and [character references](https://www.w3.org/TR/html5/syntax.html#syntax-charref), but the text must not contain an [ambiguous ampersand](https://www.w3.org/TR/html5/syntax.html#syntax-ambiguous-ampersand). There are also [further restrictions](https://www.w3.org/TR/html5/syntax.html#cdata-rcdata-restrictions) described below.

[Foreign elements](https://www.w3.org/TR/html5/syntax.html#foreign-elements) whose start tag is marked as self-closing can't have any contents (since, again, as there's no end tag, no content can be put between the start tag and the end tag). [Foreign elements](https://www.w3.org/TR/html5/syntax.html#foreign-elements) whose start tag is *not* marked as self-closing can have [text](https://www.w3.org/TR/html5/syntax.html#syntax-text), [character references](https://www.w3.org/TR/html5/syntax.html#syntax-charref), [CDATA sections](https://www.w3.org/TR/html5/syntax.html#syntax-cdata), other [elements](https://www.w3.org/TR/html5/syntax.html#syntax-elements), and [comments](https://www.w3.org/TR/html5/syntax.html#syntax-comments), but the text must not contain the character "<" (U+003C) or an [ambiguous ampersand](https://www.w3.org/TR/html5/syntax.html#syntax-ambiguous-ampersand).

[Normal elements](https://www.w3.org/TR/html5/syntax.html#normal-elements) can have [text](https://www.w3.org/TR/html5/syntax.html#syntax-text), [character references](https://www.w3.org/TR/html5/syntax.html#syntax-charref), other [elements](https://www.w3.org/TR/html5/syntax.html#syntax-elements), and [comments](https://www.w3.org/TR/html5/syntax.html#syntax-comments), but the text must not contain the character "<" (U+003C) or an [ambiguous ampersand](https://www.w3.org/TR/html5/syntax.html#syntax-ambiguous-ampersand). Some [normal elements](https://www.w3.org/TR/html5/syntax.html#normal-elements) also have [yet more restrictions](https://www.w3.org/TR/html5/syntax.html#element-restrictions) on what content they are allowed to hold, beyond the restrictions imposed by the content model and those described in this paragraph. Those restrictions are described below.

Tags contain a *tag name*, giving the element's name. HTML elements all have names that only use [alphanumeric ASCII characters](https://www.w3.org/TR/html5/infrastructure.html#alphanumeric-ascii-characters). In the HTML syntax, tag names, even those for [foreign elements](https://www.w3.org/TR/html5/syntax.html#foreign-elements), may be written with any mix of lower- and uppercase letters that, when converted to all-lowercase, matches the element's tag name; tag names are case-insensitive.
